# pushup-bot

An SMS bot for running a group pushup challenge. Members text in their daily rep
counts; moderators receive recap leaderboard images by MMS.

## How it works

- Members text their rep count to the Twilio number (e.g. `50`, `just did 50`,
  `knocked out 50 pushups`). The first integer in the message is logged as that
  day's reps. Texting again the same day overwrites the previous count.
- The bot never replies. Unrecognized messages are silently ignored.
- Moderators receive the weekly recap automatically and can trigger recaps
  on demand.

## people.txt

`people.txt` is the single source of truth for who is in the challenge. It is
re-read on every webhook call and every report, so edits take effect without a
redeploy.

Format:

```
[MEMBERS]
First Last | +1XXXXXXXXXX

[MODERATORS]
First Last | +1XXXXXXXXXX
```

- Lines starting with `#` are comments; blank lines are ignored.
- **Members** are tracked for reps and appear on the leaderboard.
- **Moderators** receive recap images and can trigger reports / admin
  corrections.
- A person can be listed in both sections (they log reps *and* get reports).

To add someone, add a line under the appropriate section and save the file.

## Reports

Recap images are 900x1400 leaderboard graphics generated by
`image_generator.py` and delivered as MMS to each moderator individually.
Members never receive images.

### Scoring

Points are floored and capped at 150:

- 1-100 reps: 1 point per rep
- 101-150 reps: 0.5 points per rep
- 151+ reps: 0.25 points per rep

### Moderator commands

Texted from a number listed in `[MODERATORS]`:

- `send daily report` / `display daily report` — sends today's daily recap to
  all moderators.
- `send weekly report` / `display weekly report` — sends the current week's
  recap to all moderators.
- `admin correct <firstname> <reps>` — sets a member's reps for today.
- `admin correct <firstname> <reps> <MMDD>` — sets a member's reps for a
  specific date.

Reports are generated and sent asynchronously, so the webhook returns
immediately and Twilio never times out.

### Weekly schedule

Every Monday at 8:50 AM Eastern, the bot sends the previous complete week's
(Mon-Sun) recap to all moderators. There is no automatic daily send — daily
recaps are on-demand only.

## Environment variables

- `TWILIO_ACCOUNT_SID`
- `TWILIO_AUTH_TOKEN`
- `TWILIO_PHONE_NUMBER`
- `CLOUDINARY_URL`

## Running locally

```
pip install -r requirements.txt
python image_generator.py   # writes /tmp/preview_daily.png and /tmp/preview_weekly.png
python app.py               # starts the Flask app on port 5000
```

The SQLite database lives at `/data/pushups.db`.
